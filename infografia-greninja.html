STORED PROCEDURES

Un jutsu preparado de antemano: encapsula la lógica de negocio, se ejecuta con precisión repetible y mantiene la base de datos protegida de ataques directos.

PROYECTO · UPLA — GRADOS Y TÍTULOS DE PREGRADO
🛡️ Seguridad
♻️ Reutilización
🚀 Rendimiento

1. ¿Qué es una Stored Procedure?

Conjunto de instrucciones SQL almacenadas en la base de datos, ejecutables bajo demanda, con parámetros de entrada, salida o entrada/salida.

CREATE PROCEDURE nombre_procedimiento
  @parametro1 tipo_dato,
  @parametro2 tipo_dato = valor_default
AS
BEGIN
  -- Lógica SQL
END

2. Componentes

  • Parámetros: reciben valores de entrada o devuelven resultados
  • Cuerpo: contiene la lógica SQL a ejecutar
  • Salida: puede devolver conjuntos de resultados o valores escalares
  • Módulo: se almacena en la base de datos para su reutilización
Stored Procedures en GradosTitulos

4.1 SP_RegistrarTramite

Registra un nuevo trámite en el sistema.

CREATE PROCEDURE Tramite.SP_RegistrarTramite
  @IdPersona INT, @IdTipoTramite INT, @IdPlan INT,
  @Observacion NVARCHAR(300) = NULL,
  @IdTramite INT OUTPUT
AS
BEGIN
  SET NOCOUNT ON;
  INSERT INTO Tramite.Tramite (IdPersona, IdTipoTramite, IdPlan,
    Observacion, FechaRegistro, Estado)
  VALUES (@IdPersona, @IdTipoTramite, @IdPlan, @Observacion,
    SYSDATETIME(), 'REGISTRADO');
  SET @IdTramite = SCOPE_IDENTITY();
END
Ejecución de ejemplo

 @Observacion=N'Solicitud de Bachiller', @IdTramite=@IdTramite OUTPUT;
SELECT @IdTramite AS IdTramiteGenerado;

4.3 SP_ObtenerTramitesPorPersona

Obtiene todos los trámites de un estudiante.

CREATE PROCEDURE Tramite.SP_ObtenerTramitesPorPersona
  @IdPersona INT
AS
BEGIN
  SET NOCOUNT ON;
  SELECT T.IdTramite, TT.NombreTramite, P.CodigoPlan,
    T.FechaRegistro, T.Estado, T.Observacion
  FROM Tramite.Tramite T
  INNER JOIN Catalogo.TipoTramite TT ON TT.IdTipoTramite=T.IdTipoTramite
  INNER JOIN Academico.PlanEstudio P ON P.IdPlan=T.IdPlan
  WHERE T.IdPersona = @IdPersona
  ORDER BY T.FechaRegistro DESC;
END

4.4 SP_ValidarRequisitosTramite

Valida si el estudiante cumple los requisitos del trámite.

CREATE PROCEDURE Tramite.SP_ValidarRequisitosTramite
  @IdTramite INT
AS
BEGIN
  SET NOCOUNT ON;
  SELECT CASE WHEN (SELECT H.PromedioPonderado
    FROM Academico.HistorialAcademico H
    WHERE H.IdPersona = (SELECT IdPersona FROM Tramite.Tramite
     WHERE IdTramite=@IdTramite) AND H.PromedioPonderado >= 13)
   THEN CAST(1 AS BIT) ELSE CAST(0 AS BIT) END AS CumpleRequisitos;
END
Resultado simulado — CumpleRequisitos: 1

4.5 SP_ObtenerActasPorSustentacion

Devuelve las actas de jurado de una sustentación.

CREATE PROCEDURE Evaluacion.SP_ObtenerActasPorSustentacion
  @IdSustentacion INT
AS
BEGIN
  SET NOCOUNT ON;
  SELECT A.IdActa, MJ.Cargo, P.Nombres, P.Apellidos,
    A.Observacion, A.Aprobado
  FROM Evaluacion.Acta A
  INNER JOIN Evaluacion.MiembroJurado MJ ON MJ.IdMiembroJurado=A.IdMiembroJurado
  INNER JOIN Academico.Persona P ON P.IdPersona=MJ.IdPersona
  WHERE A.IdSustentacion = @IdSustentacion
  ORDER BY MJ.Cargo;
END

FUNCIONES Y FUNCIONES DE TABLA

Movimientos precisos, reutilizables en cualquier combate: escalares que devuelven un solo valor, y de tabla que devuelven conjuntos completos de filas listas para usarse en cualquier consulta.

1. ¿Qué es una función en SQL Azure?

Objeto que devuelve un valor escalar o una tabla; se usa en consultas, vistas, procedimientos y otras funciones.

  • Escalares: devuelven un solo valor
  • De tabla en línea: devuelven un conjunto de filas
  • De tabla con múltiples instrucciones: lógica compleja + tabla

2. Escalar: ObtenerEdad

CREATE OR ALTER FUNCTION dbo.ObtenerEdad(@FechaNacimiento DATE)
RETURNS INT
AS
BEGIN
  RETURN DATEDIFF(YEAR, @FechaNacimiento, GETDATE());
END
Uso
SELECT NombreCompleto, dbo.ObtenerEdad(FechaNacimiento) AS Edad
FROM Academico.Persona;

3. Escalar: FormatearDNI

CREATE OR ALTER FUNCTION dbo.FormatearDNI(@DNI VARCHAR(8))
RETURNS VARCHAR(12)
AS
BEGIN
  RETURN STUFF(STUFF(@DNI,3,0,'-'),6,0,'-');
END

5. Multi-instrucción: fn_EstudiantesEstadoAcademico

Devuelve estudiantes con su promedio y estado académico calculado.

CREATE OR ALTER FUNCTION dbo.fn_EstudiantesEstadoAcademico()
RETURNS @Tabla TABLE (
  IdEstudiante INT, NombreCompleto NVARCHAR(150),
  Promedio DECIMAL(5,2), Estado NVARCHAR(20)
)
AS
BEGIN
  INSERT INTO @Tabla
  SELECT p.IdPersona, p.NombreCompleto,
    dbo.CalcularPromedioNotas(p.IdPersona) AS Promedio,
    CASE WHEN dbo.CalcularPromedioNotas(p.IdPersona) >= 14
      THEN 'Aprobado'
      WHEN dbo.CalcularPromedioNotas(p.IdPersona) >= 11
      THEN 'En Observación' ELSE 'Desaprobado' END
  FROM Academico.Persona p
  WHERE p.TipoPersona = 'ESTUDIANTE';
  RETURN;
END

6. fn_EstudiantesPorFacultad (en línea)

CREATE FUNCTION dbo.fn_EstudiantesPorFacultad(@IdFacultad INT)
RETURNS TABLE
AS
RETURN (
  SELECT E.IdEstudiante, E.DNI, E.Nombres, E.Apellidos,
    F.NombreFacultad, E.FechaIngreso
  FROM Academico.Estudiante E
  INNER JOIN Academico.Facultad F ON E.IdFacultad=F.IdFacultad
  WHERE E.IdFacultad = @IdFacultad AND E.Activo = 1
);

7. fn_HistorialTramitesEstudiante (multi-instrucción)

CREATE FUNCTION dbo.fn_HistorialTramitesEstudiante(@IdEstudiante INT)
RETURNS @Historial TABLE (
  IdTramite INT, NombreTramite NVARCHAR(100),
  FechaTramite DATETIME, Estado NVARCHAR(20), Observacion NVARCHAR(200)
)
AS
BEGIN
  INSERT INTO @Historial
  SELECT T.IdTramite, TT.NombreTramite, T.FechaTramite, T.Estado, T.Observacion
  FROM Tramite.Tramite T
  INNER JOIN Catalogo.TipoTramite TT ON T.IdTipoTramite=TT.IdTipoTramite
  WHERE T.IdEstudiante = @IdEstudiante
  ORDER BY T.FechaTramite DESC;
  RETURN;
END
